## zz PWM 채널 분배 전략

라파 4의 하드웨어 PWM 채널은 2개 (PWM0, PWM1). HW PWM 가능 핀에 의도적으로 배치:

| 신호 | GPIO | HW 채널 | 동시 활성 가능? |
|---|:---:|:---:|:---:|
| 좌 전진 (BTS#1 RPWM) | 18 | PWM0 | 우 전진과 동시 활성 |
| 우 전진 (BTS#2 RPWM) | 19 | PWM1 | 좌 전진과 동시 활성 |
| 좌 후진 (BTS#1 LPWM) | 12 | PWM0 | 우 후진과 동시 활성 |
| 우 후진 (BTS#2 LPWM) | 13 | PWM1 | 좌 후진과 동시 활성 |

같은 BTS의 전진/후진은 절대 동시 활성 안 됨 → 채널 공유 충돌 자연스럽게 회피.

**운용:** 초기엔 4개 다 SW PWM (pigpio)으로 통일. 나중에 필요 시 HW PWM 모드로 결선 변경 없이 코드만 수정.

## 🔌 보드별 결선

### PCA9685 (서보 제어)

| PCA9685 핀 | 신호 | 연결 |
|---|---|---|
| VCC | 로직 전원 | 라파 5V (분기) |
| GND | 로직 GND | 라파 GND (분기) |
| SDA | I2C 데이터 | 라파 GPIO 2 (Pin 3) |
| SCL | I2C 클럭 | 라파 GPIO 3 (Pin 5) |
| OE | (안 씀, 비워둠) | - |
| V+ (초록 터미널) | 서보 전원 | **7.4V 배터리 +** |
| V- (초록 터미널) | 서보 GND | **7.4V 배터리 -** |
| 채널 0~5 | 서보 출력 | 변형 서보 6개 |
| 채널 6~9 | 서보 출력 | 조향 서보 4개 (FL, FR, RL, RR) |
| 채널 10~15 | (안 씀) | - |

I2C 주소: **0x40**

### BTS#1 (좌측 DC 모터 3개)

| BTS#1 핀 | 신호 | 연결 |
|---|---|---|
| VCC | 로직 전원 | 라파 5V (분기) |
| GND | 로직 GND | 라파 GND (분기) |
| RPWM | 전진 PWM | 라파 GPIO 18 (Pin 12) |
| LPWM | 후진 PWM | 라파 GPIO 12 (Pin 32) |
| R_EN | 전진 Enable | 라파 GPIO 23 (Pin 16, Y분기) |
| L_EN | 후진 Enable | 라파 GPIO 23 (Pin 16, Y분기) |
| R_IS, L_IS | (안 씀) | - |

| B- (초록 터미널) | 모터 전원 - | **11.1V 배터리 -** |
| M+ (초록 터미널) | 모터 출력 + | 좌측 모터 3개 M1 (납땜 묶음) |
| M- (초록 터미널) | 모터 출력 - | 좌측 모터 3개 M2 (납땜 묶음) |

## 🔧 DC 모터 케이블 (6선)

각 모터의 6선 라벨:

| 라벨 | 의미 | 연결 |
|:---:|---|---|
| M1 | 모터 코일 + | BTS 묶음 → M+ |
| M2 | 모터 코일 - | BTS 묶음 → M- |
| VCC | 엔코더 전원 | 라파 3.3V (좌/우 대표만) |
| GND | 엔코더 GND | 라파 GND (좌/우 대표만) |
| C1 | 엔코더 A상 | 라파 GPIO (좌/우 대표만) |
| C2 | 엔코더 B상 | 라파 GPIO (좌/우 대표만) |

**엔코더 살리는 모터:** 좌측 가운데 (ML), 우측 가운데 (MR)
**엔코더 비활성 모터:** FL, RL, FR, RR — VCC/GND/C1/C2 끝부분 절연테이프 마감

## 🛠 자작 케이블 목록

| # | 케이블 | 가닥 | 용도 |
|:---:|---|:---:|---|
| 1 | 5V 분기 | 4-way (1→3) | 라파 5V → PCA, BTS#1, BTS#2 VCC |
| 2 | GND 분기 | 4-way (1→3) | 라파 GND → PCA, BTS#1, BTS#2 GND |
| 3 | 엔코더 VCC Y분기 | 3-way (1→2) | 라파 3.3V → 좌/우 엔코더 VCC |
| 4 | 엔코더 GND Y분기 | 3-way (1→2) | 라파 GND → 좌/우 엔코더 GND |
| 5 | BTS#1 EN Y분기 | 3-way (1→2) | 라파 GPIO 23 → R_EN+L_EN |
| 6 | BTS#2 EN Y분기 | 3-way (1→2) | 라파 GPIO 24 → R_EN+L_EN |
| 7 | 좌측 M+ 묶음 | 4가닥 납땜 | 좌측 모터 3개 M1 → 굵은 전선 → BTS#1 M+ |
| 8 | 좌측 M- 묶음 | 4가닥 납땜 | 좌측 모터 3개 M2 → 굵은 전선 → BTS#1 M- |
| 9 | 우측 M+ 묶음 | 4가닥 납땜 | 우측 모터 3개 M1 → 굵은 전선 → BTS#2 M+ |
| 10 | 우측 M- 묶음 | 4가닥 납땜 | 우측 모터 3개 M2 → 굵은 전선 → BTS#2 M- |

## ⚡ 전원 흐름

```
[11.1V 배터리 (3S Li-ion, 2600mAh)]
  ├─→ BTS#1 B+/B-  →  좌측 모터 3개 (병렬)
  └─→ BTS#2 B+/B-  →  우측 모터 3개 (병렬)

[7.4V 배터리 (2S Li-ion, 7.0V 정도로 충전)]
  └─→ PCA9685 V+/V-  →  서보 10개

[5V 보조배터리 (USB-C)]
  └─→ 라파  →  (점퍼선 분기)
                ├─→ PCA9685 VCC
                ├─→ BTS#1 VCC
                └─→ BTS#2 VCC

[공통 GND]: 위 셋 모두 라파 GND를 통해 한 점에 모임
```

## 💻 코드 상수 (Python)

```python
# 라파 GPIO (BCM 번호)
PIN_SDA = 2
PIN_SCL = 3

# 좌측 모터 (BTS#1)
PIN_L_RPWM = 18   # 좌 전진
PIN_L_LPWM = 12   # 좌 후진
PIN_L_EN   = 23

# 우측 모터 (BTS#2)
PIN_R_RPWM = 19   # 우 전진
PIN_R_LPWM = 13   # 우 후진
PIN_R_EN   = 24

# 엔코더
PIN_L_ENC_A = 17  # 좌 C1
PIN_L_ENC_B = 27  # 좌 C2
PIN_R_ENC_A = 22  # 우 C1
PIN_R_ENC_B = 5   # 우 C2

# PCA9685 채널
SERVO_TRANSFORM = list(range(0, 6))   # 0~5
SERVO_STEERING = {
    'FL': 6, 'FR': 7,
    'RL': 8, 'RR': 9,
}

# PWM 설정
PWM_FREQ_MOTOR = 1000  # 1 kHz (DC 모터)
PWM_FREQ_SERVO = 50    # 50 Hz (서보, PCA9685가 처리)

# I2C
PCA9685_ADDR = 0x40
```

## ⚠️ 안전 체크리스트

### 배터리 결선 전 필수 확인
- [ ] 멀티미터로 배터리 극성 (+/-) 재확인
- [ ] BTS7960 B+/B- 거꾸로 안 연결 (거꾸로 = 보드 즉사)
- [ ] PCA9685 V+/V- 거꾸로 안 연결
- [ ] 7.4V 배터리 만충 (8.4V) 상태 피하기 — 7.0V 정도까지 방전 후 사용
- [ ] 11.1V 배터리 라파에 직결 금지 (라파는 5V만)

### 단락 방지
- [ ] 배터리 두 단자 절대 직접 닿지 않게
- [ ] 배터리 분리 시 +/- 끝에 절연테이프 감기
- [ ] 한 번에 한 선씩 작업

### 결선 점검
- [ ] 공통 GND 확인 (라파 GND ↔ PCA9685 GND ↔ BTS GND 모두 연결됨)
- [ ] 5V/GND 점퍼선 색 거꾸로 안 꽂혔는지
- [ ] 모터 단자 나사 단단히 조였는지

### 동작 검증 순서
1. 라파 부팅 + SSH 접속
2. `sudo i2cdetect -y 1` → 0x40 확인 (PCA9685 인식)
3. 서보 1개 swing 코드 → 동작 확인
4. DC 모터 1개 작은 duty (20%) 회전 → 동작 확인
5. 점진적으로 부품 추가

## 📅 진행 단계

### Phase 1: 배터리 없이 가능한 결선 (오늘)
- 라파 ↔ PCA9685 점퍼선 (4가닥)
- 라파 ↔ BTS#1 점퍼선 (5가닥 + EN Y분기)
- DC 모터 ↔ BTS#1 (모터 케이블 절단, M1/M2만)
- 서보 ↔ PCA9685 채널 0

### Phase 2: I2C 검증
- `i2cdetect -y 1` → PCA9685 0x40 확인

### Phase 3: 충전기 확인
- 학교에서 배터리 충전기 찾기
- 충전기 모델 + 커넥터 확인
- 절단 vs 짝꿍 vs 핀빼기 결정

### Phase 4: 배터리 결선
- 7.4V → PCA9685 V+/V-
- 11.1V → BTS#1 B+/B-
- 극성 멀티미터로 재확인

### Phase 5: 동작 테스트
- 서보 swing
- DC 모터 회전 + 방향 확인
- 좌/우 일관성 (본 시연 시)

### Phase 6: BTS#2 추가 (BTS#2 도착 후)
- BTS#2 결선 (BTS#1과 대칭)
- 좌/우 동시 제어 검증

### Phase 7: 모터 6개 통합 (본 시연 결선)
- 좌측 3개 병렬 납땜 묶음
- 우측 3개 병렬 납땜 묶음
- 서보 12개 PCA9685에 다 꽂기
